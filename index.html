HTML

  • Enter your HTML notes here:
  • The head element contains information about the webpage.
  • The body element represent the visible content shown to the user.
  • HTML attributes define additional information about an element.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets (<>).
  • In seemantic HTML elements, the name of the element describe its content and the role it plays on a webpage.
  • A link(<>) element creates the connection to the external file.
  • The rel attritubte specifies the relationship between the current document and the linked document resource.
  • The href attribute specifices the location (URL) of the external resource.
  • The href attribute is using a relative path. This means the location is derived from the current location
  • href: hypertext reference

CSS

  • Enter your CSS notes here
  • CSS (Cascading Style Sheets) is a programming language used by developers to define how webpage content is presented to users.
  • Inline CSS, internal CSS style sheet, and external CSS style sheet are the three ways to style a webpage using CSS.
  • When a developer wants to style and element directly in the HTML file, they use inline CSS
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal style sheets contain CSS rules for the webpage in the head section of the HTML file.
  • The most common way to style a webpage is by using an external CSS style sheet.
  • External style sheets allow developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • A selector defines the elements or attributes to which the rules or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • A padding property adds space around the content inside an element.
  • A wild card symbol or * selector applies a rule to all the elements visible on the page
  • Selectors can be combined into a single CSS rule with a comma, allowing the declaration to affect multiple selectors.
  • Two or more elements that will share the same styling can be grouped together reducing repetition and extra lines of code.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages reusing or sharing pieces of code in order to reduce the number of lines written or maintained.
  • text-align: a positioning property, one way to center a title or heading on a webpage
  • The img(<>) selector assigns property values to any image element in the markup.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • The class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.
  • The period (.) designates an element as a class selector.
  • auto value: The value of said property is adjusted automatically according to the content or the context of the element.
  • The box-shadow CSS property adds shadow effects around an element's frame.
  • Colors can be assigned by including semantic references such as "gray" or "blue" or through hex code (hex #).

Git

  • Enter your Git notes here
  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • The git command must precede every Git statement.
  • git checkout: moves the working branch to a new branch
  • git add: adds modifications in the current working branch to the staging area
  • The staging area informs Git that changes will be added in the next commit.
  • git pull: receives a branch's modifications into the local environment
  • The origin keyword indicates the source of the pull will be in the GitHub repo.
  • It is important to always do a git pull to make sure you are on the latest version of the project before working.
