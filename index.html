HTML

  • Enter your HTML notes here:
  • The head element contains information about the webpage.
  • The body element represent the visible content shown to the user.
  • HTML attributes define additional information about an element.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets (<>).
  • In seemantic HTML elements, the name of the element describe its content and the role it plays on a webpage.
  • A link(<>) element creates the connection to the external file.
    • JS files are linked within the body(<>) element.
    • CSS files are linked within the head(<>) element.
  • The rel attritubte specifies the relationship between the current document and the linked document resource.
  • The href attribute specifices the location (URL) of the external resource.
  • The href attribute is using a relative path. This means the location is derived from the current location
  • href: hypertext reference

CSS

  • Enter your CSS notes here
  • CSS (Cascading Style Sheets) is a programming language used by developers to define how webpage content is presented to users.
  • Inline CSS, internal CSS style sheet, and external CSS style sheet are the three ways to style a webpage using CSS.
  • When a developer wants to style and element directly in the HTML file, they use inline CSS
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal style sheets contain CSS rules for the webpage in the head section of the HTML file.
  • The most common way to style a webpage is by using an external CSS style sheet.
  • External style sheets allow developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • A selector defines the elements or attributes to which the rules or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • A padding property adds space around the content inside an element.
  • A wild card symbol or * selector applies a rule to all the elements visible on the page
  • Selectors can be combined into a single CSS rule with a comma, allowing the declaration to affect multiple selectors.
  • Two or more elements that will share the same styling can be grouped together reducing repetition and extra lines of code.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages reusing or sharing pieces of code in order to reduce the number of lines written or maintained.
  • text-align: a positioning property, one way to center a title or heading on a webpage
  • The img(<>) selector assigns property values to any image element in the markup.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • The class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.
  • The period (.) designates an element as a class selector.
  • auto value: The value of said property is adjusted automatically according to the content or the context of the element.
  • The box-shadow CSS property adds shadow effects around an element's frame.
  • Colors can be assigned by including semantic references such as "gray" or "blue" or through hex code (hex #).

Git

  • Enter your Git notes here
  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • The git command must precede every Git statement.
  • git checkout: moves the working branch to a new branch
  • git add: adds modifications in the current working branch to the staging area
  • The staging area informs Git that changes will be added in the next commit.
  • git pull: receives a branch's modifications into the local environment
  • git push: push up your changes to the remote repository
  • The origin keyword indicates the source of the pull will be in the GitHub repo.
  • It is important to always do a git pull to make sure you are on the latest version of the project before working.
  • Git flow:
    • Create a branch, creating a space to work without affecting the default (main) branch. git checkout -b branch name
    • Make changes, add new files, edit existing ones, moving files to new locations, etc.. git add -A
    • Commit and push your changes to your branch, give each commit a descriptive messages to help understand what changes are contained. git commit - m "commit description"
    • Create pull request, ask collaborators for feedback on your changes. Include a summary of the changes and problems they solve. git pull origin main; git push origin "branch name"
    • Merge your pull request, onced approved merge your branch so that your changes appear on the default branch.
    • Delete your branch, indicates a work branch is complete and prevents you or others from accidentally using old branches.

JavaScript

  • Enter your JavaScript notes here
  • Variables are containers that store values.
  • In JS to declare a variable, we use the var keyword. Variable names must be unique.
  • A variable can be declared and given a value in the same line.
  • A semicolon at the end of a line indicates where a statement ends.
  • JavaScript is case sensitive. myVariable ≠ myvariable
  • String: a sequence of characters used to represent text, signified using '' or ""
  • Boolean: a logical data type that can only have the values true or false
  • True/False are special keywords that don't need quote marks.
    • A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false.
  • Array: a single variable that is used to hold a group of data.
    • Ex. Array: let myVariable = [1,'Bob','Steve',10];
    • A single array can store different types of data, such as strings, numbers, and so on.
    • The length property can be used to find the length of an array.
  • Each item in an array has a unique number attached to it, called a numeric index, that allows you to access it individually.
  • Ex: let myVariable = [1,'Bob','Steve',10]; myVariable[0] = 1
  • A for() loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A for() loop contains the following three important statements:
    • The first statement determines the starting point for our loop. Ex: var x = 0;
    • The second statement is the condition. Ex: x < shapes.length;
    • The final statement is what allows the array to iterate over each item. Ex: x++ (a shortcut for writing x+1)
    • for(var x = 0; x < shapes.length; x++)
  • Object: Everything in JavaScript is an object and can be stored in a variable.
  • Comments are snippets of text that can be added along with code.
  • You can write comments in JavaScript just as you can in CSS: /*Everything in between is a comment.*/
  • If your comment contains no line breaks, it's an option to put it behind two slashes like this: // This is a comment
  • Operator: a mathematical symbol that produces a result based on two value (or variables).
    • +: add two numbers together or combine two strings
    • -, *, /: subtraction, multiplication, division respectively, perform the same as in basic math
    • = : assignment, assigns a value to a variable
    • === :Strict equality, performs a test to see if two values are equal. It returns a true/false (Boolean) result.
    • !, !==: Not, Does-not-equal, returns the logically opposite value of what it precedes. It turns a true into a false, etc..
    • When (!, !==) is used alongside the Equality operator, the negation operator tests whether two values are not equal.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Function: a set of instructions that tells the computer how to perform a certain task
  • Functions are a way of packaging functionality that you wish to reuse.
  • Functions do not automatically execute, to execute a function we must call it in our code..
  • To create a function we use the function keyword to declare a new variable, then we give our function a unique name:
    • Ex: function helloWorld(){// code to be executed}
  • Functions often take arguments: bits of data they need to do their job.
  • Arguments: a value (primitive or object) passed as input to a function.
  • Arguments go inside the parentheses, separated by commas if there is more than one argument.
  • Event handlers: code structures that listen for activity in the browser, and run code in response
  • Null: a special value in JS that indicates the absence of an object
  • Undefined: indicates the absence of a value.
  • JavaScript is a dynamic language with dynamic types. Variables are not directly associated with any particular value type, and any variable can be assigned (and re-assigned) values of all types.
  • JS is also a weakly typed language, which means it allows implicit type conversion when an operation involves mismatched types, instead of throwing type errors.
  • Primitive types: An immutable value whose content cannot be changed without creating an entirely new value.
  • All primitive types, except null, can be tested by the typeof operator. === null is used to test for null.
  • BigInt values represent numeric values which are too large to be represented by the number primitive.
  • A BigInt is created by appending n to the end of an integer or by calling the BigInt() function.
  • JavaScript strings are immutable once a string is created, it is not possible to modify it.
  • Symbol: a unique and immutable primitive value and may be used as the key of an Object property
  • The purpose of symbols is to create unique property keys that are guaranteed not to clash with keys from other code.
  • Identifier: a sequence of characters in the code that identifies a variable, function, or property
  • We can use the console.log() method to output a message to the web console by adding an argument.
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
